# 🎓 Student Affairs & Dashboard System

An interactive, responsive, and data-driven management system designed for educational institutions to streamline the administration of students, courses, instructors, and employees. This project demonstrates advanced frontend architecture, clean state handling, and seamless dynamic integration with a mock RESTful API server.

## ✨ Key Features

* **Full CRUD Operations:** Perform seamless Create, Read, Update, and Delete operations across all data models (Students, Courses, Instructors, Employees) dynamically.
* **Interactive UI/UX:** Built using dynamic Modals (Pop-up windows) for a modern, fluid data entry experience without irritating page reloads.
* **Smart Data Filtering:** Implemented instant client-side Search and alphabetical Sorting functionalities to ease navigation within large tables.
* **Robust Pagination:** Optimized for handling larger datasets smoothly using a custom-built pagination workflow to display fixed rows per page.
* **Centralized API Architecture:** Structured utilizing a modular API helper service layer utilizing `async/await`, structural cloning (`...`), and modern JavaScript `fetch` paradigms.

---

## 🛠️ Tech Stack & Tools

* **Frontend Layout:** HTML5, CSS3, Modern JavaScript (ES6+ Concepts)
* **Styling & Responsiveness:** Flexbox, Grid, Custom CSS Variables, Adaptive Media Queries
* **Backend Emulation (REST API):** JSON Server (Mock Database)
* **Network & State Management:** Fetch API, Async/Await asynchronous handling, Service Layer Pattern
* **Version Control:** Git & GitHub
